Troubleshooting
If you are having difficulty with some aspect of using Transcribe! then please first make sure you have read the relevant parts of the Help as you may well find the answer there. That means:
  • For basic use of Transcribe! : the Overview, and Getting Started.
  • The Help for the specific area you are having a problem with.
  • The FAQ.

If something strange is happening then there may be additional information in the System Info command on the Help menu. Run this command, and see what it says. There may be technical information there which won't mean anything to you but which might mean something to me. Also if you have been seeing an error message pop up, then its text will also appear in System Info so copying from there is the easiest way to report what it said to me.

If you think there is a bug in Transcribe! then please check that you are running the latest version, and update if you aren't. Use the "Check for Updates" command on the Help menu.

Errors reported by the OS

If Windows itself has errors to report about Transcribe! or anything else, then you may be able to see error messages as follows:
Go to Start Menu - Windows Administrative Tools - Event Viewer, and run it.
In Event Viewer select "Windows Logs" in the left column and click "Application".
Then look at the list of events, for those described as "Error" and whose timestamp corresponds to when the problem happened. Click on it and see what it says.

If there is any problem with the location Transcribe! is using for temporary files then you can override this by setting the environment variable TMPDIR to the full name of the folder to use for temp files (without a terminating separator).

Problems with playback

No sound / "Play" button is disabled

First, have you successfully loaded a sound file? The wavy black line of its profile should be visible if you have, see the picture here for comparison. If you have not loaded the sound file then read Getting Started for instructions.

Can't load sound file

Check here to see if it is of a type supported by Transcribe!. Maybe the file is faulty - will it load and play into other software? What software?

Waveform profile is displayed but still no sound

Press the Play button. Does the play position indicator (vertical black line) move across the waveform and the scroll bar? If so then Transcribe! thinks it is playing... maybe the speakers aren't plugged in or something. Transcribe! does also have a volume control, see Audio Effects, make sure it is not at zero.

Check Transcribe!'s selection of output device. This is in Preferences - Playback.

Your sound card may have options for controlling what signals get sent to what output, so check this. You may find controls (often called a "mixer") for your sound card in "Start Menu" - "Settings" (the gear icon) - "System" - "Sound", or check the sound card documentation.

Drop-outs (brief silent gaps interrupting playback)

Is any other program running at the same time and possibly using the disk heavily? If so, trying stopping that program.

Also see here for advice about this one.

Sound card or MIDI is busy

If the sound card or MIDI input is reported as busy then the thing is to try to figure out who's using it. Run "Task Manager" to see a list of the programs running. Right-click the taskbar at the bottom of the screen and select "Task Manager" there.
Then look at the list of applications running. Are any of them sound-related? Close down the one which is using the sound card or MIDI and try again.

Problems with Video

If Transcribe! won't load or play some video file then first make sure it plays ok in other software on your computer.

If a video is playing in a jerky way or takes a long time to get synchronised with the audio then try using the Export Video command to create a copy. The copy will be in a format which Transcribe! may be able to do a better job of displaying.

See here in the Help for more information about video in Transcribe!, and see https://www.seventhstring.com/xscribe/video.html on the website.

Problems Recording

By far the commonest problem is that you must select the appropriate source to record from. If the VU meter in the "Record new sound file" dialog is not jumping up and down with the music then probably the recording input selection is not set correctly. See here for more information about how to fix this.

Make sure you are recording to a disk which is writable (not a CD) and which has sufficient free space.

The comments above about busy sound cards can also apply to recording.

Can any other software on your machine record successfully?
