Commands that require parameters, which can only be used for Automation.

FxMonoSetFromText(text): Set mix from text supplied.
     Parameter: text specifying the mix. For discussion, see below.
Summary: Active 0/1, XscWide 0/1, Mix 0-200 (100 is centre), PhaseReverse 0/1
Example: "1,0,100,1" sets Karaoke mode (phase reverse)

FxEqSetFromText(text): Set EQ from text supplied.
     Parameter: text specifying the EQ. For discussion, see below.
All dB figures can have one decimal place and are in the range from -80 to +20
Summary: Active 0/1, XscWide 0/1, Overall gain dB, then slider settings. The slider settings themselves are a list of 31 dB numbers separated by colons.
Example: "1,0,0.0,2.0:7.5:9.0:7.5:2.0:0.5:0.0:0.0:0.0:0.5:2.5:5.0:5.5:5.0:2.5:0.5:0.0:0.0:-0.5:-2.5:-8.0:-12.5:-8.0:-2.5:-0.5:0.0:0.0:-1.0:-3.0:-7.5:-10.0" sets a complex EQ profile. Note the distinction between the comma separators and the colon separators.

FxTuningSetFromText(text): Set tuning from text supplied.
     Parameter: text specifying the tuning. For discussion, see below.
Summary: Active 0/1, XscWide 0/1, Octaves -2 to +2, Semitones -12 to +12, Cents -60 to +60, Analog lock 0/1, Tuning note pitch 48 to 72 (MIDI notenum), Tuning note volume 0 to 100, Play tuning note 0/1
Example: "1,0,0,3,9,0,60,50,0" sets pitch to 3 semitones and 9 cents sharp.

TuningOctaveSet(int): Set tuning octaves.
     Parameter: Octaves -2 to +2.

TuningSemitoneSet(int): Set tuning semitones.
     Parameter: Semitones -12 to +12.

TuningSemitoneIncrement(int): Increment tuning semitones.
     Parameter: amount to add, any whole number though the result after adding will be restricted to -12 to +12.

TuningCentSet(int): Set tuning cents.
     Parameter: Cents -60 to +60.

TuningCentIncrement(int): Increment tuning cents.
     Parameter: amount to add, any whole number though the result after adding will be restricted to -60 to +60.

FxTranspositionSetFromText(text): Set transposition from text supplied.
     Parameter: text specifying the transposition. For discussion, see below.
Summary: Active 0/1, XscWide 0/1, Octaves -2 to +2, Semitones -12 to +12, Cents -60 to +60
Example: "1,0,-1,3,0" sets transposition for Alto Saxophone.

TranspositionOctaveSet(int): Set transposition octaves.
     Parameter: Octaves -2 to +2.

TranspositionSemitoneSet(int): Set transposition semitones.
     Parameter: Semitones -12 to +12.

TranspositionCentSet(int): Set transposition cents.
     Parameter: Cents -60 to +60.

FxSpeedSetFromText(text): Set speed from text supplied.
     Parameter: text specifying the speed. For discussion, see below.
Summary: Active 0/1, XscWide 0/1, Speed 5000 - 200000 in thousandths of a percent (i.e. 5% to 200%), Analog lock 0/1
Example: "1,0,50000,0" sets half speed.

SpeedSet(double): Set speed.
     Parameter: Speed 0.05 to 2.0 meaning 5% speed to double speed. Up to five decimal places are accepted.

SpeedIncrement(double): Increment speed.
     Parameter: amount to add, any number but the result after adding will be restricted to 0.05 to 2.0. Up to five decimal places are accepted.

FxMiscSetFromText(text): Set misc Fx (volume) from text supplied.
     Parameter: text specifying the volume. For discussion, see below.
Summary: Active 0/1, XscWide 0/1, Volume 0 to 100 (50 is standard, no change). Volume settings are always "Active" and never "Transcription-wide".
Example: "1,0,60" sets volume to 60.

VolumeSet(double): Set volume.
     Parameter: Volume 0 to 100 (50 is normal, no change).

VolumeIncrement(double): Increment volume.
     Parameter: amount to add, any number though the result after adding will be restricted to 0 to 100.

The six Fx...SetFromText commands above are intended to allow the setting of all Fx controls, since some are not accessible through commands otherwise. For instance there would otherwise be no way of setting the EQ sliders from a command. This method is clumsy but better than nothing. The text you use consists of values (numbers) separated by commas, with no spaces. Where a value is described as "0/1" it means that "0" will be interpreted as "no", and "1" as "yes".

The text used is the same as Transcribe! stores in transcription (xsc) files to record Fx settings, so if you want to decide what is the appropriate text to set a specific EQ profile then one way of doing it would be to set up that profile in the Audio Effects dialog in the usual way, then save the transcription and load the xsc file into a text editor. You will find a line saying e.g.:

FX_EQ,1,0,0.0,2.0:7.5:9.0:7.5:2.0:0.5:0.0:0.0:0.0:0.5:2.5:5.0:5.5:5.0:2.5:0.5:0.0:0.0:-0.5:-2.5:-8.0:-12.5:-8.0:-2.5:-0.5:0.0:0.0:-1.0:-3.0:-7.5:-10.0

Remove the initial "FX_EQ," off the front and the remainder is the text which will set that EQ profile when passed to the FxEqSetFromText command.

All of these start with two "0/1" vaues indicating whether the effect is "Active" (as opposed to Bypass) and whether it is "Transcription wide" ("apply to all views of this transcription" - only relevant if you have "Split" the view).
